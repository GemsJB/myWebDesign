Learning Outcomes Addressed

On successful completion of this project, you will demonstrate the ability to:

  • Design and build structured, semantic, and accessible webpages using HTML5
  • Style a fully responsive, multi-page website using CSS3 (Flexbox, Grid, media queries, animations)
  • Implement interactive behaviour and DOM manipulation using JavaScript
  • Apply Bootstrap 5 components and the responsive grid system in a real project
  • Manage and publish the project using Git version control and GitHub Pages or Netlify
  • Apply UI/UX principles and meet WCAG 2.1 accessibility criteria
  • Test, debug, and optimise a front-end application across browsers and devices
  • Present and defend design and technical decisions to an academic audience
4

Technical Requirements

🏗 HTML5 Structure
  • Minimum 4 pages (or equivalent single-page sections)
  • Semantic tags throughout: header, nav, main, section, footer
  • At least one accessible HTML form with validation attributes
  • All images include descriptive alt text
  • Valid HTML — passes W3C Validator with no errors
🎨 CSS3 Styling
  • External stylesheet(s) only — no inline styles
  • CSS custom properties (variables) for colours and typography
  • Flexbox or Grid used for all major layout sections
  • At least one keyframe animation or CSS transition
  • Media queries for mobile, tablet, and desktop breakpoints
⚡ JavaScript Interactivity
  • Minimum 3 interactive features using vanilla JavaScript
  • DOM manipulation — dynamically update content or styles
  • At least one event listener (click, submit, keyup, etc.)
  • Client-side form validation with user feedback messages
  • No jQuery — use modern ES6+ syntax
📱 Bootstrap & Responsiveness
  • Bootstrap 5 integrated via CDN or npm
  • Bootstrap grid system used for at least 2 page sections
  • At least 3 Bootstrap components (navbar, cards, modal, etc.)
  • Fully responsive — tested at 320px, 768px, and 1280px+ widths
🔧 Git, Deployment & Accessibility

Git repository with a minimum of 10 meaningful commits showing incremental development  ·  Project deployed live on GitHub Pages or Netlify  ·  WCAG 2.1 AA: sufficient colour contrast, keyboard navigation, ARIA roles where appropriate  ·  Tested on Chrome, Firefox, and mobile viewport

Academic Integrity Policy

All submitted work must be your own original creation. The following constitute academic misconduct under KCA University Academic Regulations:

  • Copying code from another student or submitting another person's project as your own
  • Submitting AI-generated code without acknowledgement or demonstrable understanding
  • Copying templates or themes from the internet without significant original modification and attribution
  • Sharing your source code with other students before the submission deadline
  • Fabricating browser testing results, commit history, or accessibility audit findings
📋 STUDENT DECLARATION — sign at submission

"I declare that this project is my own original work. All code was written by me, and any external resources (libraries, icons, fonts, images) are properly attributed in my README and project report. I understand that plagiarism may result in a grade of zero and disciplinary action in accordance with KCA University regulations."

Suggested Project Timeline

WEEK 10
Planning & Design
Select your project option; get Option H approval if needed. Produce wireframes or mockups for all pages, define your colour palette and typography, and plan your Bootstrap grid structure.
WEEK 10–11
HTML & CSS Development
Build all HTML pages with semantic structure. Apply external CSS with Flexbox/Grid layouts, CSS variables, and responsive media queries. Integrate Bootstrap grid and components. Begin Git commits.
WEEK 11
JavaScript & Interactivity
Add all JavaScript features: DOM manipulation, event listeners, and form validation. Test interactivity across browsers. Continue committing to GitHub with meaningful messages.
WEEK 11–12
Testing, Accessibility & Deployment
Run the W3C Validator, check colour contrast, test keyboard navigation, and audit with browser DevTools. Deploy to GitHub Pages or Netlify. Test the live URL on mobile and desktop.
WEEK 12
Submission & Presentation
Submit all deliverables via the Student Portal by 11:59 PM. Present your live site with a 5–7 minute demo, take part in Q&A, and complete the peer code review exercise in class.
